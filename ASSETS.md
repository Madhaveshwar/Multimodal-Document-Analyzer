# GitHub Assets Guide

This project is designed to present well on GitHub and in recruiter reviews. Use the following assets to give the repository a polished public-facing look.

## Project Banner Suggestion

Recommended banner specs:

| Property | Recommendation |
|---|---|
| Size | 1600 × 500 px |
| Format | PNG or WebP |
| Style | Dark, modern, enterprise-ready |
| Visual theme | OCR, document intelligence, RAG, workflow automation |
| Palette | Deep slate, cyan, teal, and soft amber accents |
| Text | "Multimodal AI SaaS Platform" or "Document Intelligence for Production Teams" |

Suggested composition:

- Left: product title and a short value proposition
- Center: subtle document, camera, and AI nodes connected with flowing lines
- Right: a clean UI mockup or dashboard preview
- Footer: small badges for Streamlit, Docker, LangSmith, ChromaDB, and SQLite

## GIF / Demo Section

Recommended demo media:

| File | Purpose |
|---|---|
| `docs/demo/workflow-demo.gif` | Short loop showing upload → OCR → chat → export |
| `docs/demo/rag-answer-demo.gif` | Source citations and grounded answers |
| `docs/demo/deployment-demo.gif` | Container start-up and health check |

Tips for a strong demo:

- Keep the loop under 20 seconds.
- Show a real document and a real answer with citations.
- Use a dark background and a clean cursor path.
- Prefer one cohesive story over many disconnected UI moments.

## README Placement Guidance

Place the banner near the top of the README, followed by badges and a short hero paragraph. Put screenshots after the feature highlights and before the deployment section so the narrative flows from value proposition to proof.
