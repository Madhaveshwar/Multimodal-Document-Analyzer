# Contributing

Thank you for taking the time to improve this project.

## How to Contribute

1. Fork the repository and create a feature branch.
2. Keep changes focused and easy to review.
3. Update the relevant documentation when behavior or deployment steps change.
4. Run the app locally and verify the affected workflow before opening a pull request.
5. Prefer small, well-scoped commits that are easy to understand.

## Review Expectations

- Document user-visible changes clearly.
- Keep security-sensitive defaults conservative.
- Avoid adding new dependencies unless they materially improve the release.
- Preserve compatibility with the existing Streamlit workflow unless a change is explicitly intended to replace it.

## Good First Contribution Areas

- Documentation improvements
- Screenshot or demo asset creation
- UX polish and accessibility improvements
- Additional deployment guidance
- Test coverage for critical workflows

## Pull Request Checklist

- [ ] README updated if the user experience changed
- [ ] Architecture or deployment docs updated if needed
- [ ] Secrets are not committed
- [ ] Temporary files are excluded
- [ ] Manual validation completed for the touched workflow
